  SLOPE → NOISE-FLOOR MECHANISM ANALYSIS
  ResShift + BicubicDownsample(4) — existing data only, no new GPU passes

────────────────────────────────────────────────────────────────────────
  SECTION 1: ANALYTIC PREDICTION
────────────────────────────────────────────────────────────────────────

  Calibration setup (exact, from layer/calibrate.py):
    stack = {x_out_i}, i=1..N   (rectified ensemble members)
    mean_x[p]   = mean_i(x_out_i[p])       (per-pixel ensemble mean)
    pred_std[p] = std_i(x_out_i[p])         (per-pixel ensemble std)
    actual_err[p] = |mean_x[p] - x_gt[p]|
    K=10 quantile bins by pred_std → (x_k, y_k) bin means
    slope  β̂ = Σ(x_k - x̄)(y_k - ȳ) / Σ(x_k - x̄)²   [OLS]

  Derivation of Var(β̂) across independent N-sample windows:

  Let μ(p) = E[x̂_j(p)], σ²(p) = Var[x̂_j(p)] across seeds.
  Since x_out_i = A⁺y + (I−A⁺A)x̂_i, the range part A⁺y is FIXED,
  so std_i(x_out_i[p]) = std_i((I−A⁺A)x̂_i[p]).

  Mean reconstruction: mean_x[p] = A⁺y[p] + (I−A⁺A)·(1/N Σ_i x̂_i[p])
  For null-space pixel p:  actual_err[p] = |(I−A⁺A)(mean_x[p] − x_gt[p])|
                                         = |bias[p] + δ[p]|
  where:
    bias[p] = (I−A⁺A)(μ(p) − x_gt[p])  [irreducible, fixed per image]
    δ[p]    = (I−A⁺A)(1/N Σ_i x̂_i[p] − μ(p))  [sampling noise ~N(0,σ²(p)/N)]

  In the regime bias(p) >> δ(p) (consistent model predictions):
    E[actual_err(p)] ≈ |bias(p)| ≡ β·pred_std(p)  (calibration slope β)
    Var_windows[actual_err(p)] ≈ Var[δ(p)] ≈ σ²(p)/N ≈ pred_std²(p)/N

  Bin k contains n_k pixels.  Under the IID PIXEL ASSUMPTION
  (independent actual_err across pixels within a bin):
    Var_windows[y_k] ≈ (1/n_k²) Σ_{p∈k} pred_std²(p)/N
                     ≈ x_k²/(n_k · N)    [since pred_std(p)≈x_k in bin k]

  OLS slope variance (heteroscedastic):
    Var(β̂) = Σ_k c_k² · Var(y_k)   where c_k = (x_k−x̄)/Σ(x_j−x̄)²
           = (1/(n_eff·N)) · Σ_k c_k² · x_k²
    where n_eff ≤ n_k accounts for pixel correlation.

  KEY PROPERTY — SCALE INVARIANCE: if all x_k → s·x_k (same distribution
  shape, rescaled), then c_k → c_k/s, Var(y_k) → s²·Var(y_k), and
  Var(β̂) is UNCHANGED.  The slope estimate's variance does not depend
  on the magnitude of predicted_std values.

  ┌─────────────────────────────────────────────────────────────────┐
  │ IID PREDICTION (null hypothesis):                               │
  │   SD(β̂) ∝ 1/√(n_eff · N)  ×  f(x_k distribution shape)        │
  │   α = 0 (slope noise floor is β-INDEPENDENT under iid pixels)  │
  └─────────────────────────────────────────────────────────────────┘

  If α > 0 empirically, the iid assumption is violated.  The only free
  parameter is n_eff — the effective number of independent pixels per bin.
  n_eff < n_k when pixel errors are spatially correlated within a window.
  If n_eff varies systematically with image type (lower for images where
  ResShift generates spatially coherent hallucinations), a correlation
  between slope and noise floor could arise through that mediating variable.
  That would be a MODEL/DATA EFFECT, not an estimator artifact.

────────────────────────────────────────────────────────────────────────
  SECTION 4: SOFT_BLOBS — DEGENERATE REGIME
────────────────────────────────────────────────────────────────────────

  soft_blobs: slope=4.3725, null_frac=0.0014 — almost no null-space energy.
  Ensemble std (predicted uncertainty) is near-zero for smooth blobs.
  The calibration bins have tiny x_k values; even small seed-to-seed
  fluctuations in the ensemble mean produce large changes in the regression
  slope (fitting a near-flat signal). This is an ILL-CONDITIONING effect.

  soft_blobs  std=0.3391  range=0.8991  (11× boardwalk std of 0.0286)
  This outlier behaviour is CONSISTENT with the IID estimator model:
  when n_k·pred_std² is tiny, Var(y_k) explodes → Var(β̂) explodes.
  It is an ESTIMATOR ARTIFACT of the degenerate-signal regime, unrelated
  to slope magnitude per se (there are low-slope images with low null that
  would show the same behaviour).

────────────────────────────────────────────────────────────────────────
  SECTION 5: VERDICT
────────────────────────────────────────────────────────────────────────

  Relative noise (std/slope) by image — β-independent would give constant:

  Image               slope       std   std/slope
  ──────────────────────────────────────────────
  wood_grain         0.2323    0.0186      0.0802
  boardwalk          0.9475    0.0286      0.0302
  soft_blobs         4.3725    0.3391      0.0776
  boy_face           3.1776    0.0511      0.0161
  girl_sad_face      2.7961    0.0374      0.0134
  wayuu_woman        1.5848    0.0471      0.0297

  If noise floor were β-proportional (α=1): std/slope would be constant.
  If β-independent (α=0): std would be constant, std/slope ∝ 1/slope.

  POWER LAW FIT RESULT: α̂ = 0.69 (full n=6, t-CI [-0.19,1.56]),  α̂ = 0.36 (excl. soft_blobs n=5, t-CI [0.07,0.64])

  The boardwalk vs. boy_face comparison (matched null_frac ≈ 0.0046-0.0049,
  the cleanest controlled pair in the dataset):
    slope ratio: 3.1776/0.9475 = 3.35×
    std ratio:   0.0511/0.0286 = 1.78×
    If α=1 (proportional): expected std ratio = 3.35× — NOT supported
    If α=0 (independent):  expected std ratio = 1.00×  — NOT supported
    If α=0.5 (sqrt):       expected std ratio = 1.83× — supported (actual 1.78×)

  CONCLUSION — MODEL/DATA EFFECT, not pure estimator artifact:

  (a) The analytic derivation shows that under iid pixels, SD(β̂) is
      β-INDEPENDENT and scale-invariant (α=0). The OLS regression estimator
      itself does NOT produce slope-proportional noise.

  (b) The data (excl. soft_blobs, n=5) give α̂=0.36, t-CI [0.07,0.64],
      excluding 0 at p<0.05 (df=3). The full fit (n=6) t-CI [-0.19,1.56]
      includes 0, with soft_blobs as a high-leverage outlier.
      The restricted fit is the more reliable signal — soft_blobs is a
      known degenerate-regime case with a separate mechanism (§4).

  (c) The matched boardwalk/boy_face pair (same null_frac) shows 1.78× std
      ratio for 3.4× slope ratio — consistent with α≈0.5, inconsistent
      with both α=0 and α=1.

  (d) soft_blobs is ill-conditioning from low signal, a separate estimator
      artifact not related to slope magnitude.

  MECHANISTIC INTERPRETATION: The slope noise floor scales as slope^α with
  α≈0.35 (sub-proportional). The OLS estimator predicts α=0 under iid
  pixels. The empirical α>0 therefore indicates that pixel reconstruction
  errors are NOT iid within bins — they are spatially correlated. The
  effective number of independent observations n_eff is lower than the raw
  pixel count n_k. Images with higher calibration slopes (faces) appear to
  have lower n_eff, suggesting ResShift generates more spatially coherent
  (correlated) hallucinations for those images. This is a MODEL/DATA EFFECT.

  CAVEAT: n=5 data points (excluding soft_blobs) spanning one order of
  magnitude in slope. The point estimate α̂=0.35 is plausible but imprecise.
  The conclusion (model effect, not estimator artifact) is supported by
  the analytic prediction AND the matched-pair evidence, not just the fit.

────────────────────────────────────────────────────────────────────────
  IMPLICATION FOR §10 DOMAIN-SHIFT SNR
────────────────────────────────────────────────────────────────────────

  The SNR calculation in §10 (natural↔faces, 9.2×) already used the
  FACES GROUP's own noise floor (boy_face N=12 range=0.1497) as the
  denominator — the side of the contrast with the HIGHER slope and HIGHER
  noise floor. This makes the SNR calculation CONSERVATIVE.

  If slope magnitude drives noise floor (slope-proportional noise floor),
  then the faces noise floor is partly inflated by faces having high slopes.
  Using that inflated denominator for the contrast PENALISES the SNR.
  The 'honest' SNR under a slope-normalised noise measure would be HIGHER,
  not lower, than the stated 9.2×.

  Concretely: natural mean slope ≈ 1.19 vs. faces mean slope ≈ 2.57.
  If noise floor ∝ slope, the natural group's noise floor is roughly
  0.0302×slope = 0.0359 for naturals, and 0.0776 for faces.
  The actual worst noise floor used was 0.1497 (boy_face). The natural
  group's noise floor (boardwalk SD=0.0286) is already captured separately.

  VERDICT: the slope-noise mechanism, even if real, makes the §10 9.2×
  SNR MORE conservative, not less. The §10 verdict ('faces slope elevation
  is 9× above worst in-contrast noise floor') stands. The mechanism does
  NOT weaken §10 — if anything it implies the true SNR is higher.

  This finding does not require editing §10. Flagged here for completeness.

  Empirical fit (n=6):             α̂ = 0.69, CI [-0.19, 1.56]
  Empirical fit (n=5, excl. soft):  α̂ = 0.36, CI [0.07, 0.64]

  N-scaling:   noisy (5-window std estimates); consistent with 1/√N within uncertainty
  §10 impact:  NONE — the SNR calculation is already conservative on the mechanism side

  (a) vs (b) verdict: MODEL/DATA EFFECT (n_eff varies with image type).
  Evidence: α=0 rejected at p<0.05 (excl. soft_blobs); boardwalk/boy_face
  matched pair consistent with α≈0.5; analytic derivation shows estimator
  predicts α=0 under iid, so α>0 implicates the data structure, not the
  estimator. The effect is sub-proportional (α≈0.35–0.5 < 1).
  To characterise precisely: n≥20 images with repeat noise-floor measurements,
  spanning slope range with null_frac matched across slope levels.
  This is a future investigation — not a blocker for §10.
