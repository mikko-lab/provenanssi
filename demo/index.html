Provenance-Aware Image Reconstruction

ResShift × BicubicDownsample — calibrated uncertainty on held-out ImageNet images

What this shows

Every AI reconstruction has two kinds of content: what the degraded input constrains, and what the model invented to fill the gaps. This demo makes that distinction visible and measurable.

Recovered

The input constrains this pixel's low-frequency energy. The model's reconstruction varies little across seeds — modest null-space contribution.

Invented

This pixel's energy is majority null-space. The input gave the model little constraint here — the fine texture is the model's fabrication.

Key distinction: "Invented" means not determined by the input — not necessarily wrong. The invented content may look exactly right; you simply cannot know from the measurement alone. That is the provenance claim. For 4× super-resolution, high-frequency detail is structurally in the null space of the downsampling operator: all images here show a mix of recovered and invented content.

Honest caveat — slope = 1.53:
The uncertainty is well-ordered: high spread reliably signals high error (r = 0.97). But the absolute scale is off by ~1.5×: the model is mildly underconfident. The worst bin has actual error ≈ 5.4× the predicted spread. Calibration passes the threshold (slope ≤ 2.0), but perfect magnitude calibration (slope = 1.0) has not been demonstrated.

Honest limitation case

Image: ILSVRC2012_val_00017068  ·  Per-image r = 0.849 (below the per-image 0.9 threshold).
85% of pixels are classified as INVENTED (highest among the three demo images). The uncertainty ordering is weaker here: in regions of complex high-frequency texture, the model's spread does not reliably rank-order its errors. The pooled 16-image r = 0.967 holds because this image is one of 16 — but it shows where the calibration is weaker.

Scope: This demo presents results for a single degradation (bicubic 4× downsampling) and a single model (ResShift, trained on ImageNet bicubic pairs). Ground truth is available because this is a synthetic evaluation: the LR input was computed from the HR ground truth. No claim is made about real camera images, manipulation detection, or any operator other than BicubicDownsample(4).

Certified: Calibration metrics verified by falsify.py --full (commit 5f99f84) — Pearson r = +0.9667, slope = 1.5301, ECE = 0.0282, IS_CALIBRATED = YES, n = 16 images × 6 samples. Thresholds fixed; not tuned.

Accessibility: All interactive elements are keyboard-operable. Colour is never the sole distinguisher: each provenance class uses colour + texture pattern + text label. Contrast ≥ 4.5:1 for text. Markup uses semantic HTML.
